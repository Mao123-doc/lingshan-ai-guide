# 语音播放改为手动触发 — 设计文档

## 问题陈述

当前 AI 回答完成后自动播放 TTS 语音，存在两个 bug：一是 TTS 响应慢导致语音迟迟不播放，二是语音经常播到一半就中断。改为手动触发播放可以彻底规避这两个问题。

## 需求 & 约束

- 去掉"AI 回答完成后自动播放语音"的行为
- 每条 AI 回复消息旁显示独立的播放按钮，用户手动点击才播放
- 点击播放时**立即播放**（后台已预生成 TTS 缓存）
- 同时只有一条消息在播放，新点一条自动停掉前一条
- 保留全局静音开关：打开时播放按钮会出声，关闭时播放按钮只有动画不出声
- 不修改 TTS 服务端代码，只改前端交互逻辑

## 推荐方案

| 决策点 | 选择 |
|---|---|
| TTS 时机 | 后台预生成（AI 回答返回后静默调 TTS 并缓存） |
| 按钮位置 | 每条 AI 消息气泡底部独立按钮 |
| 并发控制 | 新播放自动停止旧播放 |
| 全局开关 | 保留，改为"静音模式" |

## 架构草图

```
AI 回答 streaming 完成
        │
        ▼
后台静默调 /api/v1/visitor/tts ──→ 缓存 { audioBase64, visemes[], duration }
        │
        ▼
消息气泡渲染时检查：TTS 缓存好了吗？
  ├─ 好了 → 显示 ▶️ 播放按钮（可点击）
  └─ 没好在加载 → 显示 ⏳ 加载中（不可点击）

用户点击 ▶️
  ├─ 停止当前正在播放的消息（如有）
  ├─ 检查全局静音开关
  │   ├─ 未静音 → 播放 audio + 数字人口型动画 + 按钮变⏸️
  │   └─ 静音   → 只播放数字人口型动画（不播 audio）+ 按钮变⏸️
  └─ 播放完毕 → 按钮恢复 ▶️，数字人回 idle

用户点击 ⏸️（播放中）
  └─ 停止播放，按钮恢复 ▶️
```

## 关键状态设计

```typescript
// 每条消息的 TTS 缓存状态
type TTSStatus = 'idle' | 'loading' | 'ready' | 'playing';
// idle: 还没开始加载
// loading: 正在请求 TTS
// ready: TTS 已缓存，可以播放
// playing: 正在播放中

// 消息扩展字段
interface ChatMessage {
  // ...原有字段
  ttsStatus: TTSStatus;
  ttsAudioBase64: string | null;
  ttsVisemes: Array<{ time_ms: number; viseme_id: number }> | null;
  ttsDuration: number | null;
}
```

## 实现步骤

1. **QAPage.tsx — 去掉自动播放**
   - 删除 `speakFullAnswer(fullAnswer)` 调用（SSE done 事件处理中）
   - 删除 fallback 中的 `speakFullAnswer` 调用
   - 删除 `cleanForTTS`、`findBestVoice`、`fetchServerTTS`、`playServerAudio`、`speakFullAnswer` 中仅用于自动播放的逻辑

2. **QAPage.tsx — 新增 TTS 预缓存逻辑**
   - AI 回答完成后（SSE done 事件），启动后台 TTS 请求
   - 将返回的 `{ audioBase64, visemes, duration }` 缓存到对应消息对象上
   - 消息 `ttsStatus` 从 `loading` → `ready`

3. **QAPage.tsx — 新增播放控制状态机**
   - 维护 `activePlayingMsgId: string | null` 和 `globalMuted: boolean`
   - 点击播放：停止旧播放 → 创建 Audio → 同步 viseme 动画 → 设置 `ttsStatus = 'playing'`
   - 播放结束/手动停止：清理 Audio + viseme 定时器，`ttsStatus` 恢复 `ready`
   - 点击另一条消息的播放按钮：自动停止当前播放的消息

4. **QAPage.tsx — 新增播放按钮 UI**
   - 每条 AI 消息气泡底部右侧，渲染播放按钮
   - 4 种状态：
     - `loading`：`<LoadingOutlined spin />` 灰色不可点
     - `ready`：`<CaretRightOutlined />` 蓝色可点击
     - `playing`：`<PauseOutlined />` 绿色可点击
     - `idle`：不显示按钮（或显示灰色不可点）
   - 当前正在播放的消息按钮高亮，其余恢复普通态

5. **QAPage.tsx — 全局静音开关改造**
   - 保留右上角按钮和 `ttsEnabled` 状态
   - 逻辑改为：`ttsEnabled=false` 时，点播放按钮只驱动 viseme 动画不创建 Audio

6. **QAPage.css — 样式**
   - `.msg-play-btn` 播放按钮样式
   - 播放中脉冲动画
   - 加载中旋转动画

## 删除的代码清单

- `cleanForTTS()` 函数（TTS 服务端已处理特殊字符）
- `bestVoiceRef` 及 `findBestVoice()`（不再用浏览器 TTS）
- `fetchServerTTS()` 的自动调用逻辑（保留函数本身用于预缓存）
- `playServerAudio()` 的自动触发（改为手动触发）
- `speakFullAnswer()` 整个函数
- `serverAudioRef` 的自动播放逻辑
- SSE done 事件处理中的 `speakFullAnswer(fullAnswer)` 调用
