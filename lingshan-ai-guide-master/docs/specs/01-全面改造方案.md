# 灵山胜境 AI 数字人导游系统 — 全面改造方案

> 基于赛题需求与现有代码的深度审计，制定本改造方案。
> **核心决策：统一后端到 Python（FastAPI），保留 React 前端。**

## 二、推荐方案：Python 统一后端 + Agnes AI 多模态

### 为什么选 Python？

| 维度 | Python | Node.js/TS |
|------|--------|-----------|
| 嵌入模型 | ✅ sentence-transformers 原生 | ❌ 需 JS 移植，质量差 |
| 向量数据库 | ✅ ChromaDB 一流支持 | ⚠️ Chroma JS SDK 功能不全 |
| RAG 框架 | ✅ LangChain/LlamaIndex 一流 | ⚠️ LangChain.js 是二等公民 |
| ASR/TTS | ✅ Whisper/CosyVoice 原生 | ❌ 需调 CLI |
| AI 生态 | ✅ HuggingFace 全生态 | ❌ 几乎为零 |

当前 Python 代码（`vectordb/`、`config.py`）质量不错，直接激活即可。

### 为什么选 Agnes AI？

- `agnes-2.0-flash`：1M 上下文、免费、OpenAI 兼容格式
- `agnes-image-2.1-flash`：满足多模态（游客拍照识景）
- 零成本接入，API 格式与现有 OpenAI 兼容代码一致

## 四、详细实施计划

### 阶段一：基础设施搭建（预计 0.5 天）

**目标：后端能跑起来，前端能连通。**

1. **清理项目结构**
   - 删除 `backend/src/`（TS 后端，全部假数据）
   - 删除 `backend/package.json`、`backend/tsconfig.json`
   - 保留 `backend/api/`、`backend/vectordb/`、`backend/config.py`
   - 新建规范的 Python 项目结构

2. **统一配置**
   - 更新 `.env` 添加 `AGNES_API_KEY`、`AGNES_BASE_URL`、`AGNES_MODEL`
   - 更新 `config.py` 支持 Agnes provider

3. **启动 FastAPI 服务**
   - 创建 `backend/app.py` 主入口
   - 挂载静态文件（前端 dist）
   - CORS + 基础中间件

4. **前端适配**
   - 更新 `api.ts` 指向新 API 路径
   - 确保前端 dev server proxy 指向 Python 后端

### 阶段二：知识库 + RAG 核心（预计 1.5 天）

**目标：真正的向量检索 RAG，问答准确率 ≥ 90%。**

1. **知识库索引脚本**
   - 读取 `data/raw/knowledge_guide.txt` + `knowledge_dataset.txt`
   - 用 `ChineseTextChunker`（已有）分块
   - 用 `BAAI/bge-large-zh-v1.5`（已有）生成嵌入
   - 写入 ChromaDB（已有 `chroma_client.py`）

2. **RAG 检索管线**
   - 查询嵌入 → ChromaDB 相似度检索（top_k=5）
   - 构建 Prompt：系统指令 + 检索结果 + 对话历史 + 用户问题
   - 调用 Agnes-2.0-Flash 生成回答

3. **RAG 准确性测试框架**
   - 从知识库中编写 50 道标准测试题
   - 每道题有标准答案
   - 自动评测脚本：语义相似度 + 关键词命中
   - 目标：准确率 ≥ 90%

4. **重排优化（可选）**
   - 引入 BGE Reranker 对检索结果重排

### 阶段三：Agnes AI 多模态集成（预计 1 天）

**目标：满足赛题"至少 1 个多模态大模型"要求。**

1. **文本对话（Agnes-2.0-Flash）**
   - OpenAI 兼容客户端 `openai` 包
   - 流式 SSE 输出
   - 1M 上下文 → 可容纳整个知识库

2. **视觉识别（Agnes-Image）**
   - 游客拍照 → 上传图片 → Agnes-Image 识别景点
   - 返回景点名称 + 自动触达该景点的详细讲解

3. **语音识别（Faster-Whisper）**
   - 服务端 ASR 兜底（浏览器 Web Speech API 不支持时）
   - 流式识别 WebSocket 通道

4. **语音合成（Edge-TTS Python 库）**
   - 用 Python 版 `edge-tts` 替代 CLI 调用
   - 音频缓存（相同文本不重复生成）
   - 流式返回 base64

### 阶段四：数据库与真实数据（预计 1 天）

**目标：所有管理端数据来自真实数据库而非硬编码。**

1. **数据库模型（SQLAlchemy + SQLite）**
   - `users` — 管理员账号
   - `conversations` — 对话会话
   - `messages` — 对话消息（含情感标签）
   - `feedback` — 游客反馈
   - `knowledge_documents` — 上传的知识文档
   - `daily_stats` — 每日统计快照

2. **对话日志收集**
   - 每次问答自动记录：时间、问题、回答、延迟、情感标签
   - 用于管理端实时统计

3. **情感分析管线**
   - 使用 Agnes-2.0-Flash 对每条对话做情感分类
   - 分类：正面/中性/负面 + 情感强度
   - 存储到 `messages` 表

### 阶段五：管理后台真实化（预计 1 天）

**目标：管理端所有图表数据来自数据库查询。**

1. **数据大屏 API**
   - 从数据库查询今日/本周/本月服务人次
   - 实时满意度计算（来自 feedback 表）
   - 时段分布统计
   - 热门问答 Top 10

2. **感受度报告 API**
   - 情感分布统计
   - 话题聚类（高频词分析）
   - AI 生成文字总结（调用 Agnes）

3. **知识库管理 API**
   - 文档上传 → 自动分块 → 自动建索引 → 更新 ChromaDB
   - 索引状态查看
   - RAG 准确性在线测试

4. **数字人配置 API**
   - 外观参数实际持久化
   - 声音参数实际生效于 TTS

### 阶段六：数字人交互优化（预计 0.5 天）

**目标：提升数字人自然度。**

1. **口型同步优化**
   - TTS 返回音频时附带时间戳
   - 前端基于字符位置 + 音频时间做口型映射
   - 至少 5 种嘴型（张、闭、半张、微笑、O型）

2. **情感表情联动**
   - 基于 Agnes 情感分析结果驱动表情
   - 正面 → smile，负面 → 关切表情，中性 → 正常讲解

3. **语音流式播放**
   - TTS 音频流式传输
   - 边接收边播放（替代当前全部下载后播放）

## 五、技术选型总结

| 组件 | 选择 | 理由 |
|------|------|------|
| **后端框架** | FastAPI | 异步、高性能、自动 OpenAPI 文档 |
| **数据库** | SQLite + SQLAlchemy | 零配置、开发友好、满足赛题需求 |
| **向量数据库** | ChromaDB | 已有代码、轻量、嵌入式 |
| **嵌入模型** | BAAI/bge-large-zh-v1.5 | 中文 SOTA、已有代码 |
| **大模型** | Agnes-2.0-Flash | 免费、1M 上下文、多模态、OpenAI 兼容 |
| **ASR** | Faster-Whisper (base) + Web Speech API | 双引擎兜底 |
| **TTS** | Edge-TTS (Python 库) | 免费、中文自然度高 |
| **分块** | LangChain RecursiveCharacterTextSplitter | 已有代码、中文优化 |
| **前端** | React 18 + Ant Design + ECharts | 保留现有、无需重写 |

## 八、风险与应对

| 风险 | 应对 |
|------|------|
| Agnes API 不稳定/限流 | 保留本地关键词 fallback；缓存常见问答 |
